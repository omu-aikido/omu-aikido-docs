## Git の利用方法

プロジェクトは[GitHub](https://github.com/omu-aikido/omu-aikido-app)で公開されている Public Repo です。
開発に参加する際は、以下の手順に従ってください。

### 手順

import { Steps } from "@astrojs/starlight/components";
import { Tabs, TabItem } from "@astrojs/starlight/components";
import { Aside } from "@astrojs/starlight/components";

<Steps>
1. リポジトリのクローン
    <Tabs>
      <TabItem label="HTTPS">
        ```bash
        git clone 'https://github.com/omu-aikido/omu-aikido-app.git'
        ```
      </TabItem>
      <TabItem label="GitHub CLI">
        ```bash
        gh repo clone omu-aikido/omu-aikido-app
        ```
      </TabItem>
    </Tabs>

1. 作業ブランチの作成

    ```bash
    git checkout -b feature/your-feature-name
    ```

    ここで、`your-feature-name` は作業内容を表す名前をつけてください。  
    また、ブランチ名は `作業分類/` で始まるようにしてください。  
    作業分類は以下のいずれかを選択してください。

    - feat: 新機能の追加
    - fix: バグの修正
    - ref: リファクタリング
    - test: テストの追加・修正
    - chore: その他の作業
    - hotfix: 緊急の修正

    <Aside>
        作業に関連したissueがすでに存在している場合は、VSCodeのGitHub拡張機能からブランチを作成することができます。\
        その場合は、自動生成のブランチ名をそのまま利用しても構いません。
    </Aside>

1. 作業を行います。

    作業を行う前に、最新のリモートリポジトリの変更を取り込んでください。

    ```bash
    git pull origin main
    ```

1. 変更のコミット

    ```bash
    git add .
    git commit -m "Add feature or fix bug"
    ```

1. リモートへのプッシュ

    ```bash
    git push origin feature/your-feature-name
    ```

1. プルリクエストを作成し、コードレビュー後にマージしてください。

</Steps>
