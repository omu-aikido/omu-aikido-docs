## GitHub の基本

GitHub は、プロジェクトの共同作業およびコード管理のためのプラットフォームであり、ユーザーがリポジトリを共有しながら実装や修正を行える環境を提供しています。

1.  リポジトリのフォーク

    フォークとは、公開されているリポジトリを自分のアカウントに複製することです。
    元リポジトリの完全なコピーを自動的に作成します。
    これにより、直接元リポジトリを変更することなく、安心して実験的な変更や新機能の実装を試みることが可能となります。

    ユーザーは Web インターフェース上でフォーク元の更新状況を確認しながら、自分のコピーに適宜反映させることで、常に最新の状態を保つことができるので
    安全かつ効率的にプロジェクトに参加できます。

2.  ブランチ管理とプルリクエスト作成

    GitHub 上での効率的な共同作業を実現するために、各機能や修正ごとに独自のブランチを作成し、その変更を統合するためのプルリクエストを提出する仕組みがあります。

    新たに作成したブランチ上で変更を加えると、元のコードに直接影響を与えず、個別のフィードバックを受けながら安全に開発を進めることができます。

    さらに、プルリクエスト提出時には、変更の背景や目的、テストの過程、期待する結果などを詳細に記述することで
    リポジトリの貢献者はどのような変更が行われたのかを把握できます。

3.  プルリクエストのレビューとマージ

    提出されたプルリクエストは、他のチームメンバーによってレビューされます。\
    この段階では、コードの品質や安全性、システム全体への影響が検証され、必要に応じて修正や追加の説明が求められるます。

    レビュープロセスを通じて、最終的なマージに向けた承認を得た後にのみ変更が本体に統合されるため、
    プロジェクトの品質を維持しながら、安全かつ効果的な開発が行うことができます。

4.  Issue の活用

    プロジェクトで見つかったバグや改善点、新機能の提案などは、Issue を通じて管理されます。\
    作成された Issue はそのままタスクとして扱われ、Pull Request やコミットとリンクさせることで、作業の進捗状況や関連する議論を追跡することができます。

5.  Discussions の活用

    Discussions は、コミュニティのメンバー間で幅広い議論やアイデアの共有を促進するためのフォーラム型の機能です。

    Q&A セッションや新機能の提案、プロジェクトの方針に関する議論など、Issue では扱いにくかったり、Issue を提出するほどではないようなトピックについても、
    自由に投稿することができます。
